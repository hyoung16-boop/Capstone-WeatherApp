[라즈베리파이서버를 이용한 안드로이드기반 날씨앱]

1. 프로젝트 개요
본 프로젝트인 'WeatherApp'은 사용자의 위치 정보를 기반으로 실시간 기상 정보를 제공하는 안드로이드 애플리케이션입니다. 
단순한 날씨 정보 제공을 넘어, 사용자 경험을 극대화하기 위해 상황별 맞춤 옷차림 추천, 인공지능 기반의 날씨 요약 브리핑, 
실시간 도로 CCTV 영상 확인 기능을 통합하였습니다. 또한, 사용자가 설정한 시간에 기상 정보를 분석하여 능동적으로 알림을 제공하는 스마트 알림 시스템을 구현함으로써, 
사용자가 날씨 변화에 선제적으로 대응할 수 있도록 돕는 것을 목표로 합니다.

2. 핵심 기술 및 구현 특징
본 프로젝트는 안드로이드의 최신 기술 트렌드를 적극 반영하여 개발되었습니다.

(1) 선언형 UI 프레임워크인 Jetpack Compose를 전면 도입하였습니다. 기존의 XML 기반 레이아웃 방식에서 벗어나, 
코틀린 코드만으로 UI를 구성함으로써 코드의 가독성을 높이고 컴포넌트의 재사용성을 확보하였습니다.

(2) MVVM(Model-View-ViewModel) 아키텍처와 Repository 패턴을 적용하여 데이터 로직과 UI 로직을  분리하였습니다. 
이를 통해 앱의 유지보수성을 높이고 테스트가 용이한 구조를 구축하였습니다.

(3) 상황 인지형 스마트 알고리즘을 탑재하였습니다. 체감 온도와 바람, 강수 여부를 종합적으로 분석하여 최적의 옷차림을 추천하는 알고리즘과, 
현재 기상 상태와 예보 데이터를 결합하여 사용자 친화적인 문장으로 날씨를 브리핑하는 요약 알고리즘을 자체 구현하였습니다.

(4)  백그라운드 프로세싱 최적화입니다. WorkManager를 활용하여 앱이 실행 중이지 않은 상태에서도 정해진 시간에 
정확히 기상 데이터를 수집하고 알림을 발송하며, 이 과정에서 배터리 소모를 최소화하도록 설계하였습니다.

3. 기술 스택
(1) 개발 언어: Kotlin 100%
(2) 사용자 인터페이스(UI): Jetpack Compose, Material Design 3
(3) 아키텍처: MVVM, Clean Architecture Concepts
(4) 비동기 처리: Kotlin Coroutines, Flow
(5) 의존성 주입(DI): Hilt (Dagger)
(6) 네트워크 통신: Retrofit2, OkHttp3, Gson
(7) 로컬 데이터베이스: Room Database
(8) 미디어 재생: Media3 (ExoPlayer)
(9) 위치 서비스: Google Play Services Location

4. 데이터베이스 설계
본 프로젝트는 앱의 오프라인 사용성과 데이터 지속성을 위해 로컬 데이터베이스인 Room을 사용하였으며, 주요 엔티티 설계는 다음과 같습니다.

(1) WeatherCacheEntity (테이블명: weather_cache)
사용자의 위치별 날씨 데이터를 캐싱하여 네트워크 연결이 불안정한 상황에서도 정보를 제공합니다. 
주요 필드로는 현재 기온, 체감 온도, 습도, 풍속 등의 기상 데이터와 함께 시간별 및 주간 예보 데이터를 JSON 문자열 형태로 직렬화하여 저장합니다. 
또한, 데이터의 유효성을 관리하기 위해 마지막 업데이트 시간과 해당 위치의 위도, 경도 정보를 포함합니다.

(2) AlarmEntity (테이블명: alarms)
사용자가 설정한 알람 정보를 저장합니다. 시(hour)와 분(minute)을 기본 키로 사용하지 않고 별도의 고유 ID를 부여하여 관리합니다. 
반복 알람을 위한 요일 목록(days)과 특정 날짜 알람을 위한 날짜 필드(selectedDate)를 포함하며, 
알람의 활성화 여부를 나타내는 플래그(isEnabled)를 통해 사용자의 설정을 유지합니다.

5. 기능 명세서
(1) 홈 화면 및 실시간 날씨
사용자의 현재 위치를 자동으로 파악하거나 특정 지역을 검색하여 해당 지역의 실시간 기온, 체감 온도, 미세먼지 농도, 자외선 지수 등 상세 정보를 제공합니다.

(2) 스마트 옷차림 추천
현재의 체감 온도와 강수, 강풍 여부를 분석하여 7단계의 온도 구간에 따른 최적의 옷차림을 추천합니다. 
특히 여름철에 두꺼운 외투가 추천되지 않도록 하는 등 계절적 특성을 반영한 방어 로직이 적용되어 있습니다.

(3) 날씨 요약 브리핑
단순한 수치 나열이 아닌, "겨울이지만 봄처럼 포근해요", "칼바람 때문에 체감온도가 낮아요"와 같이 
사용자가 직관적으로 날씨를 이해할 수 있는 자연어 형태의 요약 문구를 생성하여 제공합니다.

(4) CCTV 영상 확인
사용자가 조회 중인 지역 주변의 도로 CCTV 정보를 공공 데이터 API를 통해 가져와 실시간 영상으로 재생합니다. 
사용자의 현재 위치와 CCTV 사이의 거리를 계산하여 표시함으로써 정보의 유용성을 높였습니다.

(5) 스마트 알림
사용자가 지정한 시각에 자동으로 최신 날씨 정보를 백그라운드에서 갱신하여 알림으로 발송합니다. 
비나 눈 소식이 있을 경우 우선적으로 경고 메시지를 포함하며, 아침 출근 준비 시 유용한 날씨 브리핑 정보를 제공합니다.

6. 실행 방법
(1) 프로젝트 빌드
Android Studio Koala Feature Drop 버전 이상을 권장합니다. 프로젝트 루트 디렉토리에서 Gradle Sync를 수행하여 
의존성을 다운로드한 후, 'Run' 버튼을 클릭하거나 터미널에서 './gradlew installDebug' 명령어를 실행하여 빌드합니다.

(2) API 키 설정
본 프로젝트는 기상청 단기예보 조회 서비스 API와 공공데이터포털의 CCTV API를 사용합니다. 
local.properties 파일에 해당 API 키들이 정의되어 있어야 정상적인 데이터 조회가 가능합니다.

(3) 권한 허용
앱 최초 실행 시 위치 권한(ACCESS_FINE_LOCATION)과 알림 권한(POST_NOTIFICATIONS) 요청 팝업이 표시되며, 
정상적인 서비스 이용을 위해 이를 허용해야 합니다.
